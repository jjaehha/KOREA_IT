    데이터 요청 API

    동기(sync) 와 비동기(async)

    동기
        앞에서 수행한 작업이 모두 끝나야지만 다음 작업을 실행하는 것
        자바스크립트는 기본적으로 동기적으로 동작한다(앞에있는 함수가 완전히 끝나야지만 다음작업 실행)

    비동기 함수
        앞에서 수행한 작업이 모두 끝나지 않더라도 다음 작업을 실행하는 것
        조금 더 효율적으로 작업을 완료할 수 있다
        (특히 데이터를 전송하는 API 같은 경우에는 비동기적으로 동작하도록 만들어 놓았다)

    자바스크립트에서의 비동기 처리
    1. 콜백함수의 활용
    2. Promise 객체의 등장
    3. async / awiat 키워드

    Promisㄷ 객체
        데이터를 요청하면 시간이 조금 걸릴지라도 데이터 요청에 대한 결과를 받게된다
        그러니까 아직 데이터를 받아오지 않더라도 다른 작업을 비동기적으로 먼저 수행해라

        상태
            fulfilled
                데이터 요청이 완료되었고, 요청에대한 응답이 완전히 정상적으로 끝난 상태
            pending
                데이터 요청이 완료 되었고, 요청에 대한 응답을 아직 받지 못한 상태
            rejected
                데이터 요청이 완료되었고, 요청에 대한 응답이 비정상적으로 끝난 상태
                (요청이 잘못되었거나, 권한이 없거나, ...)

        비동기적으로 동작하게 만들어주는 함수들은 결과가 Promise 객체가 return 되도록 되어있다
        비동기 함수가 실행되면 아직 pending 상태더라고 다음 코드가 실행된다

        Promise 객체 안에 들어있는 then() 함수 :
            인자로 프로미스객체가 pending상태에서 벗어나면 실행시켜줄 함수를 써야한다
            이때 then 안에 인자로 넘겨주는 함수의 매개변수에는 해당 프로미스객체의
            PromiseResult 속에 들어있는 값이 전달된다
